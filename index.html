Two Main Types of Data Structures

üìè Linear Data Structures

Data arranged in a line, one after another.

  • Array
  • Stack
  • Queue
  • Linked List

Think of it like a line of people waiting.

üå≥ Non-Linear Data Structures

Data arranged like a tree or network, not in a line.

  • Tree
  • Graph

Learn Each Structure

üì¶ Arrays (The Simplest Structure)

An array is like a numbered list. Each item has an index so you can find it quickly.

index: 0 1 2 3 value: Dell HP Lenovo MacBook

‚ú® Why Arrays are Useful:

  • Easy access: Get any item instantly using its index
  • Good for collections: Store similar items together

üìö Stack (LIFO)

LIFO = Last In First Out

Think of a stack of plates: you add plates on top, and you remove from the top. You never remove the bottom plate first!

üåç Real-Life Examples:

  • Undo button (Ctrl+Z): Last action you did is the first one undone
  • Browser back button: Last page you visited is shown first
‚ùì Quick Question

‚úÖ Yes! Asus was last in, so it's first out (LIFO)!

üö∂ Queue (FIFO)

FIFO = First In First Out

Like waiting in line at a restaurant: the first person in line is served first, and the last person waits the longest.

üìä Example:

If laptops enter a queue like this: (1) Dell, (2) HP, (3) Asus

‚Üí Dell comes out first because it was first in line!

‚õìÔ∏è Linked List (Pointer-Based)

A linked list is like a chain. Each element (called a node) has:

  1. Data (e.g., "Dell")
  2. A pointer that tells you where the next node is
[Dell] ‚Üí [HP] ‚Üí [Asus] ‚Üí [Lenovo] ‚Üí NULL

Difference from Arrays:

Array

  • Stored consecutively in memory
  • Fast access by index
  • Fixed size

Linked List

  • Stored anywhere in memory
  • Must follow links one by one
  • Flexible size

‚úÖ Linked Lists are Great When:

  • You want to easily insert/delete items
  • You don't know how many items you'll have
